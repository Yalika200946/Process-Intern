## Missing links and duplicated calculations found

1. **Duplicated HX_CONFIG** (src/domain/config.py vs src/features/heat_duty.py) — the exact failure mode `src/domain/config.py`'s own docstring warns about ("a stale independently maintained copy... already caused a bug once") has recurred. Confirmed live-affecting E112C's Q duty calculation.
2. **Two independent network-modelling efforts inside the Claude codebase itself**: `src/network/hybrid.py` (topology reconstruction + counterfactuals, self-gated failed) and `src/network/attribution.py` (local/network compensation-ratio diagnostics, correctly conservative) solve overlapping problems with no cross-reference. Neither is wired into the dashboard.
3. **Two disagreeing rankings** feed the dashboard: `engineering_priority_score` (C23, the stated primary design) and `hx_ranking.json`'s `priority_score` (C24, source pipeline not identified in this pass) — the dashboard's own tooltip admits they disagree.
4. **`src/schemas.py`'s contract-validation framework is built but never called** from `pipeline/run_all.py` or any production notebook — a governance gate that exists in code but not in the actual execution path.
5. **No missing link found between C09 (fouling rate) and C19-C21 (PHM)** — this connection is clean; PHM correctly reuses the same curve-fitting library (`src/models/fouling_curves.py`) rather than re-implementing it.
6. **Economics (C13) correctly reads measured cleaning-event data first**, falling back to a calibrated model estimate (CF16) only for the 11/16 HX lacking measured events — a well-designed, non-duplicated measured-first pattern.

## Dashboard components demonstrating genuinely advanced capability (per task instruction not to downgrade valid features)

- **HX drill-down** (6+ synchronized charts per HX: sawtooth, predicted-vs-actual Q, temps, flows, custom multi-variable overlay, fouling-rate-per-run table, audit history) is a materially richer per-asset view than anything found in the Codex output set (which currently produces static matplotlib contact-sheets, not an interactive drill-down).
- **Scenario analysis**: the cleaning-simulation slider (date-based, showing CIT-loss reset and cumulative THB saved) is a working, interactive version of exactly the "date-based cleaning benefit" and "cleaning reset scenarios" SmartPM-inspired capabilities (see the gap matrix).
- **Evidence and traceability**: the Evidence&Confidence tab's explicit beats-persistence column and reliability badges are a stronger, more usable implementation of "evidence panel" than a static CSV registry.
- **Cleaning ranking + economics + optimization**: fully interactive and wired to a real, reviewed-clean optimizer (C16), not just a proposal document.

None of these are recommended for removal or downgrade in this audit; the correctness gating from the earlier Codex reconciliation still applies to what these components *claim*, but the mechanisms themselves are sound engineering.
