# CPHT Analytics — Docs Entry Point

**Status:** CURRENT

This repo analyzes crude preheat train (CPHT) fouling, cleaning
prioritization, CIT/furnace-F101 impact, and economics for Bangchak Plant 3,
and publishes an approved dataset to a dashboard. Start here.

## Where to start

1. Read `CLAUDE.md` (repo root) for the mandatory development rules —
   they override everything else.
2. Read `CURRENT_PIPELINE_MAP.md` for what the pipeline actually does today
   (producers/consumers/outputs, verified against real code, not aspiration).
3. Read `UNRESOLVED_ENGINEERING_DECISIONS.md` for what is still
   `REQUIRES_ENGINEERING_CONFIRMATION` before you treat any number as final.

## How to run it

```
python pipeline/run_all.py                # recompute in place
python pipeline/run_all.py --input new.xlsx
python pipeline/run_all.py --only 13      # single notebook, for debugging
```

See `RUN.md` (repo root) for the dashboard/backend and `DEPLOY.md` for
Docker/deployment. `pipeline/run_all.py` executes the notebooks in
`notebooks/` (numbered `01`–`16`) in dependency order, then a chain of
`pipeline/*.py` post-processors that write `dashboard/data/*.json`.

## Where things live

- **Production notebooks:** `notebooks/01_*.ipynb`–`16_*.ipynb` (flat,
  numbered — see `CURRENT_PIPELINE_MAP.md` for the exact chain).
- **EDA / diagnostic notebooks:** `notebooks/_eda_*.ipynb`,
  `notebooks/_diagnostic_*.ipynb` — not called by `run_all.py`.
- **Archived notebooks:** `notebooks/_archive_2026-07-12/`.
- **Shared engineering/modeling code:** `src/domain`, `src/features`,
  `src/models`, `src/optimization`, `src/reporting`, `src/validation` (see
  `MIGRATION_MAP.md` for what moved here from `notebooks/*.py` and why).
- **Pipeline orchestration:** `pipeline/run_all.py` (batch) and
  `backend/server.py` (dashboard's upload/recompute API).
- **Configuration:** `config/*.yaml` (limits, tag mapping, HX topology,
  data-quality rules, data contracts) — engineering limits and thresholds
  belong here, not hard-coded in notebooks.
- **Data:** external plant data lives outside the repo (`CPHT_DATA_DIR`,
  see `CURRENT_PIPELINE_MAP.md`); this repo's own `data/{raw,interim,processed}/`
  are placeholders for a future Parquet migration (currently empty,
  gitignored — the active pipeline still uses CSV, see `MIGRATION_PLAN.md`
  Phase 3).
- **Generated output:** `dashboard/data/*.json`, `outputs/*.csv`,
  `figures/**`, `models/*` — none of this is source; it's regenerated by
  `pipeline/run_all.py`.

## Top limitations right now

See `UNRESOLVED_ENGINEERING_DECISIONS.md` for the full register. Highlights:

- `Q_norm` is not one formula in this codebase (three coexisting versions —
  `Q_NORM_AUDIT.md`) and is not approved as a fouling indicator.
- E112C's topology (spare shell vs. separate position) is unresolved and
  affects two pipelines differently.
- No maintenance/cleaning log exists yet, so cleaning events are inferred,
  not confirmed (see `docs/UNRESOLVED_ENGINEERING_DECISIONS.md` §5).
- `backend/server.py` has no authentication; see §11 there.
